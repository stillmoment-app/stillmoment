
# Implementierungsplan: ios-048

Ticket: [ios-048](../ios/ios-048-typografie-newsreader-geist.md)
Erstellt: 2026-05-16

## Leitstern

**Design aus dem Handover uebernehmen ohne Over-Engineering.** Bestehende Architektur (`TypographyRole` + `FontSpec` + `ThemeTypographyModifier`) bleibt. Es kommt eine Familie-Property hinzu, die `design: .rounded`-Property faellt weg, alles andere bleibt. Keine spekulativen Abstraktionen, keine `<em>`-Infrastruktur „auf Vorrat", keine neue Italic-Rolle. Tabular Figures fuer Ziffern wird erst gebaut, wenn sich beim manuellen Test zeigt, dass Newsreader-Ziffern sichtbar springen.

## Annahmen

- **Halation-Kompensation bleibt erhalten.** Light Mode → 300, Dark Mode → 400 (eine Stufe schwerer). User-Entscheidung. Loest reales Lesbarkeitsproblem; Handoff zeigt nur einen Modus und widerspricht dem nicht.
- **Italic nur via Markup in Akzentfarbe.** User-Entscheidung. Bestehendes dekoratives `.italic()` in `RunningTimerDisplay.swift:79` wird entfernt. Keine neue `TypographyRole.italicAccent`. Wenn spaeter eine View Italic-Hervorhebungen braucht (z.B. „Lifted *Warm*"), wird das pro Stelle via SwiftUI Markdown (`Text("**Wort**")`) mit Highlight-Farbe geloest — nicht in diesem Ticket.
- **`Font.Design` (`.rounded`) entfaellt im FontSpec.** Custom Fonts haben kein `design`-Konzept; weiterhin `.rounded` mitzufuehren waere toter Code. `FontSpec` wird auf `case fixed(size, weight)` / `case dynamic(style, weight)` reduziert.
- **Neue Property `fontFamily: TypographyRole.Family`** mit zwei Cases: `display` (Newsreader) und `ui` (Geist). Mapping pro Rolle direkt aus Handoff-Tabelle. Klein, explizit, ohne weitere Abstraktion.
- **Font-Files statisch ins Bundle.** Newsreader 300/400/500/400italic (4 TTFs) + Geist 300/400/500 (3 TTFs). Aus Privacy- und Offline-Gruenden kein Laufzeit-Download. Google-Fonts-Quellen, beide unter SIL OFL 1.1 — `OFL.txt` liegt mit im Bundle.
- **Italic-Schnitt nicht durch System-Italic ersetzen.** Newsreader hat einen eigenen Italic-Schnitt im 400-Weight. SwiftUIs `.italic()`-Modifier auf einer Custom Font wuerde nur eine schraege Transformation der Roman-Glyphen erzwingen — fuer dieses Ticket ist `.italic()` aus dem RunningTimerDisplay raus, und der `400italic`-Schnitt ist fuer spaetere `<em>`-Hervorhebungen vorgehalten.
- **Tabular Figures nicht vorab einbauen.** Aktuell nutzt RunningTimerDisplay `.monospacedDigit()` als System-Modifier — der greift bei Custom Fonts nicht. Wir messen im manuellen Test, ob Newsreader Light bei der MM:SS-Zaehlung sichtbar springt. Falls ja: Mini-Helper `Font.customMonospacedDigit(name:, size:)` via `UIFontDescriptor.featureSettings([.featureIdentifier: kNumberSpacingType, .typeIdentifier: kMonospacedNumbersSelector])` fuer die vier Ziffer-Rollen. Falls nein: bleibt einfach. Diese Bewertung passiert wirklich erst beim Test, nicht spekulativ.
- **PostScript-Namen lookup ueber `Font.custom(_:size:relativeTo:)`** — funktioniert sowohl mit PostScript-Namen als auch Family-Namen. Wir verwenden die offiziellen PostScript-Namen aus den TTF-Files (z.B. `Newsreader-Light`, `Geist-Light`, …). Sie werden als Konstanten im Family-Enum hinterlegt — eine Quelle der Wahrheit, keine Magic-Strings in Views.

| API | Min. iOS | Quelle | Hinweis |
|-----|---------|--------|---------|

## Design-Entscheidungen

### 1. Family als Property auf `TypographyRole`, kein separates Mapping-Dictionary

**Trade-off:** Alternative waere ein `[TypographyRole: Family]`-Dictionary irgendwo zentral. Vorteil: Mapping an einer Stelle, leichter ueberblickbar. Nachteil: Zweite Quelle der Wahrheit neben `fontSpec`, leichter aus dem Tritt mit der Rolle.
**Entscheidung:** Property direkt am `TypographyRole` (wie `fontSpec`, `textColor`, `tracking`). Halt dieselbe Struktur, eine Stelle pro Rolle.

### 2. `FontSpec` ohne `design` statt parallel `design: Font.Design?` zu fuehren

**Trade-off:** Die `design`-Property wegzulassen ist ein kleiner Breaking Change in `FontSpec`. Alternativ koennten wir sie als ungenutztes Feld behalten, damit `TypographyTests` unveraendert bleibt.
**Entscheidung:** Wegnehmen. Ungenutzter Parameter ist Schulden-im-Wartezustand. Test-Anpassung ist marginal (eine Zeile pro `fixed/dynamic`-Assertion).

### 3. PostScript-Namen ueber Family-Enum, nicht in den Views

**Trade-off:** Views koennten direkt `Font.custom("Newsreader-Light", …)` aufrufen. Aber damit waeren Schrift-Namen ueber 28 Views verstreut.
**Entscheidung:** `Family.postScriptName(weight: italic:)` zentralisiert die Namen in einer Stelle in `Font+Theme.swift`. Views rufen weiterhin `.themeFont(.screenTitle)` auf — Migration ist Null fuer Aufrufer.

### 4. `RunningTimerDisplay`-Italic raus statt durch `<em>`-Equivalent zu ersetzen

**Trade-off:** Wir koennten das „von 15 Minuten" Label mit Highlight-Farbe + Newsreader-Italic versehen.
**Entscheidung:** Nicht in diesem Ticket. Der aktuelle Italic war dekorativ — Handoff verbietet das. Das Label bleibt bodySecondary ohne Italic, in normaler Sekundaerfarbe. Wenn der Designer spaeter eine Hervorhebung dort moechte, kommt es als separates Refinement-Ticket.

## Fachliche Szenarien

### AK-1: Display-Texte erscheinen in Newsreader

- **Gegeben:** App offen, Timer-Idle-Screen, Light Mode.
  **Wenn:** Nutzer schaut auf den Section-Title und auf die grosse Idle-Ziffer „15:00".
  **Dann:** Beide sind in Newsreader (Serif). Optische Wirkung: warme Serif statt rounder Sans.

- **Gegeben:** Player-View einer Guided Meditation laeuft.
  **Wenn:** Nutzer schaut auf Titel und Restzeit-Countdown.
  **Dann:** Titel und Countdown-Ziffer sind in Newsreader.

### AK-2: UI-Texte erscheinen in Geist

- **Gegeben:** Settings-Screen.
  **Wenn:** Nutzer schaut auf alle Settings-Zeilen.
  **Dann:** Zeilen-Label und -Beschreibung sind in Geist (Sans).

- **Gegeben:** Library-View, mit Track-Liste.
  **Wenn:** Nutzer schaut auf Track-Titel, Untertitel und Action-Buttons.
  **Dann:** Alle drei sind in Geist.

### AK-3: Default-Gewichte 300/400/500 wie im Handoff

- **Gegeben:** Section-Title und Body-Display.
  **Dann:** Weight 300 (Light) — beobachtbar an der zarten Strichstaerke.

- **Gegeben:** Library-Track-Titel.
  **Dann:** Weight 400 (Regular) — benanntes Element ist klar lesbar.

- **Gegeben:** Primary-CTA-Button (z.B. „Sitzung starten").
  **Dann:** Weight 500 (Medium) — Akzent wie im Handoff.

### AK-4: Italic-Akzent ausschliesslich fuer Hervorhebungen

- **Gegeben:** Running-Timer mit Dauer-Label „von 15 Minuten".
  **Wenn:** Nutzer schaut auf das Label.
  **Dann:** Das Label ist **nicht** kursiv (Status-quo-Italic ist entfernt). Es zeigt sich in bodySecondary-Regular-Style, ohne Italic.

### AK-5: Dynamic Type skaliert weiterhin

- **Gegeben:** User-Setting Dynamic Type auf „extraExtraLarge".
  **Wenn:** Library-Liste angezeigt.
  **Dann:** Track-Titel und Untertitel werden groesser. Listenzeilen brechen das Layout nicht (z.B. kein abgeschnittener Text, keine ueberlappenden Buttons).

- **Gegeben:** Dynamic Type auf groesster Stufe.
  **Wenn:** Timer-Idle-Screen angezeigt.
  **Dann:** Section-Title und Settings-Labels skalieren mit. Die feste Idle-Ziffer („15:00", `.timerCountdown`) skaliert **nicht** mit (heute auch nicht — Verhalten unveraendert).

### AK-6: Dark Mode mit Halation-Kompensation

- **Gegeben:** App im Dark Mode, Section-Title (Weight 300).
  **Dann:** Effective Weight ist 400 (Regular) — ein Schritt schwerer, wie bisher. Lesbarkeit bleibt mindestens auf dem heutigen Niveau.

- **Gegeben:** Dark Mode, Player-CTA (Weight 500).
  **Dann:** Effective Weight bleibt 500 — Medium und schwerer werden nicht weiter kompensiert (heutiges Verhalten).

### AK-7: Tabular Figures (Konditional)

- **Gegeben:** Running-Timer zaehlt von 09:59 → 09:58 → 09:57.
  **Wenn:** Nutzer schaut auf die Ziffer.
  **Dann:** Die Ziffer-Position springt nicht horizontal.
  - Falls Newsreader-Light bereits sub-pixel-stabil wirkt: erledigt ohne Code-Aenderung.
  - Falls sichtbar springt: Mini-Helper fuer Tabular Figures bei den vier Ziffer-Rollen (`timerCountdown`, `timerRunning`, `playerCountdown`, `dialValue`).

### AK-8 (implizit): Bestehende Tests bleiben gruen

- **Gegeben:** Bestehende Suite in `TypographyTests`.
  **Wenn:** Migration abgeschlossen.
  **Dann:** Color-Tests und Halation-Tests laufen unveraendert; Size/Weight-Tests sind auf neue `FontSpec`-Form (ohne `design`) aktualisiert.

## Reihenfolge der Akzeptanzkriterien

Optimale Reihenfolge fuer TDD-Implementierung:

1. **Vorbereitung (manuell, vor TDD):** Font-Files herunterladen, ins `Resources/Fonts/`-Verzeichnis legen, `Info.plist` um `UIAppFonts` ergaenzen, `OFL.txt` hinzufuegen. App einmal starten und in einem temporaeren View `Font.custom(...)`-Aufruf testen, um zu verifizieren, dass die PostScript-Namen stimmen (haeufiger Stolperstein). Test-Code temporaer, wird wieder geloescht.
2. **FontSpec-Umbau (AK-8):** `design`-Parameter entfernen, `Family` einfuehren, neue `fontFamily`-Property pro Rolle. Tests anpassen. Halation- und Color-Tests bleiben gruen. — Grundlage fuer alles weitere.
3. **AK-1, AK-2, AK-3 zusammen:** Family-Mapping fuer alle 27 Rollen setzen. `resolvedFont` baut `Font.custom(...)` mit `Family.postScriptName(weight:)`. Neue Tests `testDisplayRolesUseNewsreader` / `testUIRolesUseGeist`. — Damit ist die Migration durch.
4. **AK-4:** `.italic()` in `RunningTimerDisplay.swift:79` entfernen. (Kein Unit-Test noetig — visueller Effekt.)
5. **AK-5:** Manueller Test mit Dynamic Type XL/XXL — Library, Timer-Idle, Settings durchgehen.
6. **AK-6:** Manueller Test mit Dark Mode — alle Hauptscreens auf Lesbarkeit pruefen.
7. **AK-7 (konditional):** Tabular Figures messen — falls sichtbar springt, Helper einbauen; sonst weglassen.
8. **Dokumentation:** CHANGELOG, Memory-Eintrag aktualisieren.

## Risiken

| Risiko | Mitigation |
|--------|-----------|
| PostScript-Namen der TTFs weichen von den Annahmen ab. Code laeuft, Schrift fehlt aber lautlos (Fallback auf System). | Schritt 1 verifiziert die Namen mit einem temporaeren `Font.custom`-Aufruf, bevor die Migration startet. Bei Fehlschlag: PostScript-Namen aus `Font Book.app` auslesen. |
| `Font.custom(_:size:relativeTo:)` fuer Custom Fonts skaliert Dynamic Type anders als `Font.system(_:relativeTo:)`. | Manueller Test in AK-5 deckt das ab. Falls Layout-Bruch: dann nur betroffene Rolle auf `Font.custom(_:fixedSize:)` umstellen (lokale Anpassung). |
| Newsreader Light wirkt im Dark Mode trotz Halation-Kompensation (auf 400) zu duenn. | Manueller Test in AK-6. Falls problematisch: punktuell `weight` der betroffenen Rolle um eine Stufe anheben — nicht das globale Compensation-Verhalten aendern. |
| Newsreader-Ziffern springen beim Countdown. | AK-7: erst messen, dann ggf. Tabular-Figures-Helper. |
| App-Bundle waechst durch 7 TTFs (~700 KB - 1 MB). | Hinnehmbar fuer Privacy-konformes Offline-Bundling. Variable-Font-Versionen waeren kleiner, aber zusaetzliche Komplexitaet — nicht in diesem Ticket. |

## Offene Fragen

Keine. Halation und Italic sind geklaert. Tabular Figures wird im Testschritt entschieden.
