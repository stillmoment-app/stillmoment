# Implementierungsplan: ios-044

Ticket: [ios-044 ‚Äî Edit-Sheet Prefill-UI](../ios/ios-044-import-prefill-edit-sheet-ui.md)
Erstellt: 2026-05-15

## Kernidee

ios-044 ist fachlich ein Mini-Feature (Datei importieren + Metadaten editieren). Der heutige Code macht es kuenstlich kompliziert durch einen ungenutzten Override-Mechanismus (`customTeacher`/`customName`/`effectiveTeacher`/`effectiveName`). ios-043 hat ihn als Transport im Import-Pfad missbraucht ‚Äî Save liefert dadurch den Prefill- statt den User-Wert an `addMeditation` (Bug).

Der Plan loest das in zwei Phasen:

1. **Domain-Cleanup zuerst** ‚Äî Override raus, eine Wahrheit pro Feld (`teacher`/`name`). Migration beim Load (`customTeacher` ‚Üí `teacher`, dann nil). Damit faellt die ganze Modus-Asymmetrie zwischen Edit und Import weg.
2. **UI-Erweiterung auf cleanem Boden** ‚Äî Modus-Parameter, X-Button, Autocomplete-Highlight, Pflichtfeld-Validation. Keine Tricks mit `effective*` mehr.

## Annahmen

- **Persistenz-Format**: `GuidedMeditation` wird vermutlich via `Codable` als JSON persistiert; `customTeacher`/`customName` sind Optionals. Beim Migrations-Decode liest der Service die alten Felder, schreibt sie in `teacher`/`name`, persistiert dann ohne die alten Felder. Detail-Mechanik wird beim Lesen von `GuidedMeditationService.swift` bestaetigt ‚Äî falls eine eigene Persistenz-Schicht ueber `Codable` hinaus existiert, im RED-Schritt entdeckt und im Plan angepasst.
- **‚ÄûUnknown Artist"-Werte** werden NICHT migriert (sind inhaltliche Werte, kein Override-Mechanismus). Bleiben in `teacher` stehen, User kann sie via Edit-Sheet selbst korrigieren.
- **`HighlightedText`** (heute in `Presentation/Views/GuidedMeditations/`) bleibt wo es ist ‚Äî Frage 7 ‚Üí Option C. Wiederverwendung im `AutocompleteTextField` per Import.
- **X-Button-Sichtbarkeit** folgt iOS-Standard `.whileEditing` (Frage 6 ‚Üí Option B): nur sichtbar wenn Feld fokussiert UND nicht-leer.
- **Autocomplete bei leerem Feld** zeigt keinen Dropdown (Frage 5: konsistent mit Library-Suche). Auch nicht nach X-Click.
- **Modus-spezifischer Sheet-Titel und Save-Button-Text** (Fragen 2 + 4): ‚ÄûMeditation importieren" / ‚ÄûImportieren" vs. ‚ÄûMeditation bearbeiten" / ‚ÄûSpeichern".
- **File-Info-Section in beiden Modi** sichtbar (Frage 3).
- **View-API**: pragmatisch ‚Äî `meditation: GuidedMeditation` + `mode` + Closures (Frage 1 ‚Üí Option B). `EditSheetState` bleibt intern, wird aber simpler.
- **Save-Closure-Signatur** `(GuidedMeditation) -> Void` bleibt. Im Caller verzweigt nach Modus: `addMeditation` vs. `updateMeditation`. Nach Override-Cleanup ist `edited.teacher`/`edited.name` direkt der finale Wert ‚Äî kein `effectiveTeacher`-Trick mehr.

## Refactorings

1. **Override-Mechanismus aus `GuidedMeditation` entfernen** (Phase 1). Scope: Domain + Infrastructure + alle Aufrufstellen. Risiko: niedrig ‚Äî keine Funktionalitaet geht verloren (Override wurde nirgendwo gelesen ausser ueber `effective*`, was wir parallel ersetzen). Migration beim Load schuetzt vor Datenverlust. ~15 Aufrufstellen, alle trivial.

2. **`EditSheetState.applyChanges()` vereinfachen** (Phase 1). Vorher: setzt `customTeacher`/`customName`. Nachher: setzt `teacher`/`name`. Tests (`EditSheetStateTests.swift`, 21 Tests) muessen angepasst ‚Äî viele werden simpler.

3. **`AutocompleteTextField` intern auf `ClearableTextField`** (Phase 2). Internes Refactoring, oeffentliche API unveraendert.

## Fachliche Szenarien

### Migration

- Gegeben: Persistierte Library mit einem Eintrag `teacher="Tara Brach"`, `customTeacher="Jon Kabat-Zinn"`
  Wenn: App startet, `loadMeditations()` laeuft
  Dann: Eintrag wird zurueckgeschrieben mit `teacher="Jon Kabat-Zinn"`, kein `customTeacher`. Beim naechsten Laden bleibt es so.

- Gegeben: Persistierte Library mit Eintrag `teacher="Unknown Artist"`, `customTeacher=nil`
  Wenn: App startet
  Dann: Eintrag bleibt `teacher="Unknown Artist"`. Wert wird NICHT migriert (kein Override-Mechanismus involviert).

- Gegeben: Migrierte Library (zweiter App-Start nach Migration)
  Wenn: `loadMeditations()` laeuft
  Dann: Nichts wird geaendert. Idempotent.

### AK-Import: Sheet oeffnet mit Prefill

- Gegeben: Datei `bodyscan-tara_brach.mp3` ohne ID3-Tags, Library hat ‚ÄûTara Brach" als bekannten Lehrer
  Wenn: User importiert die Datei
  Dann: Edit-Sheet oeffnet im Import-Modus. Lehrer-Feld zeigt ‚ÄûTara Brach", Name-Feld zeigt ‚Äûbodyscan". Sheet-Titel ‚ÄûMeditation importieren". Save-Button ‚ÄûImportieren" enabled. Kein Autofocus.

- Gegeben: Garbage-File `d067c0ea-‚Ä¶.mp3` ohne ID3
  Wenn: User importiert
  Dann: Edit-Sheet im Import-Modus mit beiden Feldern leer. Placeholder sichtbar. **Name-Feld autofokussiert**, Tastatur ist offen. Save-Button disabled.

### AK-Import: X-Button leert Feld

- Gegeben: Sheet im Import-Modus, Lehrer-Feld zeigt ‚ÄûTara Brach", User tappt ins Lehrer-Feld
  Wenn: User tappt das X im fokussierten Lehrer-Feld
  Dann: Feld ist leer, Placeholder sichtbar, X-Button verschwindet (Feld leer), Save-Button disabled. Kein Dropdown.

- Gegeben: User hat Lehrer-Feld gerade per X geleert, Cursor steht im leeren Feld
  Wenn: User tippt ‚ÄûT"
  Dann: Dropdown oeffnet mit Lehrer-Eintraegen die ‚ÄûT" enthalten, ‚ÄûT" akzent-hervorgehoben.

### AK-Import: Save persistiert mit User-Werten (vorheriger Bug aus ios-043)

- Gegeben: Sheet im Import-Modus, Prefill schlaegt ‚ÄûTara Brach" vor, User aendert auf ‚ÄûJon Kabat-Zinn"
  Wenn: User tappt Save
  Dann: `addMeditation` wird mit `teacher="Jon Kabat-Zinn"` aufgerufen (nicht ‚ÄûTara Brach"). Eintrag in Library hat `teacher="Jon Kabat-Zinn"`. (Domain-Cleanup macht diesen Test erst sauber moeglich, weil heute `customTeacher` als Transport benutzt wurde.)

### AK-Import: Cancel verwirft, Swipe-Down auch

- Gegeben: Sheet im Import-Modus, `pendingImport` aktiv
  Wenn: User tappt Cancel
  Dann: `cancelImport()` laeuft. Security-Scope freigegeben. Keine Datei-Kopie. Library unveraendert.

- Gegeben: Sheet im Import-Modus, `pendingImport` aktiv
  Wenn: User swipt Sheet nach unten
  Dann: `onDismiss`-Closure ruft `cancelImport()`. Identisch zu Cancel.

### AK-Edit: bestehende Werte unveraendert

- Gegeben: Bestehende Meditation mit `teacher="Unknown Artist"` (nicht migriert)
  Wenn: User oeffnet Edit-Sheet, aendert nichts, tappt Save
  Dann: `updateMeditation` schreibt unveraenderte Werte. ‚ÄûUnknown Artist" bleibt erhalten.

- Gegeben: Bestehende Meditation
  Wenn: User oeffnet Edit-Sheet
  Dann: Sheet-Titel ‚ÄûMeditation bearbeiten". Save-Button ‚ÄûSpeichern". Kein Autofocus. File-Info-Section sichtbar.

### AK-Universell: Pflichtfeld

- Gegeben: Beliebiger Modus, Lehrer-Feld gefuellt, Name-Feld leer
  Wenn: User tappt aufs Sheet
  Dann: Save-Button visuell disabled, nicht tappbar.

- Gegeben: Beliebiger Modus, beide Felder gefuellt
  Wenn: User tippt Whitespace in eines der Felder bis nur Spaces drin sind
  Dann: Save-Button disabled (`.trim().isEmpty`).

### AK-Universell: Return-Taste

- Gegeben: Fokus auf Lehrer-Feld
  Wenn: User tippt Return
  Dann: Fokus springt auf Name-Feld.

- Gegeben: Fokus auf Name-Feld, beide Felder valid
  Wenn: User tippt Return
  Dann: Save-Aktion wird getriggert (gleicher Effekt wie Save-Button-Tap).

### AK-Universell: Match-Highlight

- Gegeben: Sheet (beliebiger Modus), Lehrer ‚ÄûTara Brach" und ‚ÄûJon Kabat-Zinn" in `availableTeachers`
  Wenn: User tippt ‚ÄûT" ins Lehrer-Feld
  Dann: Dropdown zeigt ‚ÄûTara Brach" mit ‚ÄûT" akzent-hervorgehoben. ‚ÄûJon Kabat-Zinn" nicht im Dropdown.
