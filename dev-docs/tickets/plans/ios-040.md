## Design-Entscheidung: PraxisEditorViewModel-Erstellung (Option B)

**Trade-off:** `TimerView` könnte `PraxisEditorViewModel` selbst erstellen wenn `audioService` auf `internal` zugänglich ist (Option A — minimal). Alternativ erstellt `TimerViewModel` es per Factory-Methode (Option B — architekturkonform).

**Entscheidung:** Option B — Factory-Methode in `TimerViewModel`.

**Warum:** Der `[weak self] -> viewModel?.updateFromPraxis(saved)` Callback ist Koordinationslogik zwischen zwei ViewModels. Diese Logik gehört ins Application-Layer, nicht in eine View. Mit Option B enthält `TimerView` keine Wiring-Logik mehr — sie übergibt nur den aktuellen Praxis-Snapshot und empfängt ein fertiges ViewModel.

## Konkrete Implementierung

### 1. `StillMomentApp` — Shared Instance + StateObject

`createTimerViewModel()` wird aus `body` heraus aufgerufen — das erzeugt bei jedem SwiftUI-Rebuild ein neues `TimerViewModel`. Zu beheben durch `@StateObject`.

```swift
// StillMomentApp.swift

@StateObject private var timerViewModel: TimerViewModel
@StateObject private var guidedListViewModel: GuidedMeditationsListViewModel

init() {
    let sharedAudioService = AudioService()
    _timerViewModel = StateObject(wrappedValue: TimerViewModel(audioService: sharedAudioService))
    _guidedListViewModel = StateObject(
        wrappedValue: GuidedMeditationsListViewModel(audioService: sharedAudioService)
    )
    // ... bestehender init-Code (fileOpenHandler, inboxHandler)
}
```

In `body`:
```swift
// Vorher:
TimerView(viewModel: self.createTimerViewModel())
GuidedMeditationsListView(navigationPath: self.$libraryPath)

// Nachher:
TimerView(viewModel: self.timerViewModel)
GuidedMeditationsListView(navigationPath: self.$libraryPath, viewModel: self.guidedListViewModel)
```

`createTimerViewModel()` und `applyLaunchArgumentSettings()` werden zu einer Methode: Launch-Arguments werden weiterhin vor dem ViewModel angewendet, aber jetzt einmalig im `init()`.

### 2. `TimerViewModel` — Factory-Methode

```swift
/// Creates a configured PraxisEditorViewModel that saves back to this TimerViewModel.
/// Uses the shared AudioService — no separate instance created.
func makePraxisEditorViewModel(praxis: Praxis) -> PraxisEditorViewModel {
    PraxisEditorViewModel(praxis: praxis, audioService: audioService) { [weak self] savedPraxis in
        self?.updateFromPraxis(savedPraxis)
    }
}
```

### 3. `TimerView` — Factory-Aufruf (2 Stellen)

// Vorher (Zeile 131 und 262):
self.editorViewModel = PraxisEditorViewModel(praxis: praxisForEditor) { [weak viewModel = self.viewModel] savedPraxis in

### 4. `GuidedMeditationsListView` — Cleanup

Der `meditationService`-Parameter wird gespeichert und genutzt wenn kein `viewModel` übergeben wird. Wenn `viewModel` gesetzt ist, wird `meditationService` ignoriert — das ist inkonsistent. Cleanup: Wenn `viewModel` übergeben wird, `meditationService` nicht mehr speichern.

```swift
init(
    navigationPath: Binding<NavigationPath> = .constant(NavigationPath()),
    viewModel: GuidedMeditationsListViewModel? = nil,
    meditationService: GuidedMeditationServiceProtocol = GuidedMeditationService(),
    settingsRepository: GuidedSettingsRepository = GuidedMeditationSettingsRepository()
) {
    _navigationPath = navigationPath
    if let viewModel {
        _viewModel = StateObject(wrappedValue: viewModel)
    } else {
        _viewModel = StateObject(wrappedValue: GuidedMeditationsListViewModel(
            meditationService: meditationService
        ))
    }
    // meditationService wird nicht mehr als Property gespeichert wenn viewModel gesetzt ist
    self.settingsRepository = settingsRepository
    _settings = State(initialValue: settingsRepository.load())
}
```

## Fachliche Szenarien

### AK: Timer läuft durch bei Tab-Wechsel

- Gegeben: Timer läuft
  Wenn: User wechselt zur Bibliothek-Tab
  Dann: Timer zählt weiter, Audio-Session bleibt aktiv

- Gegeben: Timer läuft, User ist in Bibliothek-Tab
  Wenn: User wechselt zurück zum Timer-Tab
  Dann: Timer zeigt korrekte verbleibende Zeit, kein Audio-Aussetzer

### AK: Timer läuft durch bei Lock Screen

- Gegeben: Timer läuft mit 10 Minuten
  Wenn: User sperrt den Bildschirm
  Dann: Keep-Alive-Audio läuft weiter, Gong spielt zum richtigen Zeitpunkt

### AK: Guided Meditation Preview funktioniert

- Gegeben: Bibliothek hat mindestens eine Meditation
  Wenn: User hält eine Meditation gedrückt
  Dann: Preview startet und spielt Audio ab

- Gegeben: Preview läuft
  Wenn: User lässt los
  Dann: Preview stoppt

### AK: Preview und Timer schließen sich gegenseitig aus

- Gegeben: Timer läuft
  Wenn: User startet Meditation-Preview in der Bibliothek
  Dann: AudioSessionCoordinator übergibt Session an Preview-Source, Timer-Audio pausiert korrekt

- Gegeben: Preview läuft in Bibliothek
  Wenn: User wechselt zum Timer und startet
  Dann: Preview stoppt, Timer-Session wird aktiviert

### AK: Settings-Preview funktioniert

- Gegeben: User öffnet PraxisEditor (Timer-Einstellungen)
  Wenn: User tippt auf Gong-Preview
  Dann: Gong-Sound spielt

- Gegeben: User öffnet PraxisEditor
  Wenn: User tippt auf Hintergrund-Preview
  Dann: Hintergrund-Sound spielt für 3 Sekunden

## Risiken

| Risiko | Mitigation |
|--------|-----------|
| `applyLaunchArgumentSettings()` muss vor ViewModel-Init laufen (UI-Tests) | In `StillMomentApp.init()` zuerst aufrufen, dann ViewModels erstellen |
| `@StateObject` ignoriert nach erstem Init weitere Werte — korrektes Muster zwingend | Standard SwiftUI-Pattern, identisch mit `fileOpenHandler`/`inboxHandler` im bestehenden Code |
| `GuidedMeditationsListView` wird an anderen Stellen (Tests, Previews) ohne `viewModel` instanziiert | Default-Parameter bleiben — kein Breaking Change |
