Still Moment · Kerzenschein 2.0 · Typografie 2.1

Ein kleineres, robusteres Typo-System — gebaut auf Dynamic Type.

Das aktuelle System hat 26 Styles, mischt feste pt-Werte mit semantischen iOS-Namen und bricht bei Accessibility-Einstellungen. Dieser Plan reduziert auf 10 Tokens, macht alle skalierbar nach Apples Dynamic-Type-Mechanik und definiert klar, wo wir bewusst nicht mitskalieren — damit nichts überläuft.

Geräte-Range · iPhone SE 2022 (375pt) → iPhone 17 Pro Max (440pt) Dynamic Type · xSmall → AX5 Familien · Newsreader 300 / 400 · Geist 400 / 500

Drei Prinzipien

Tragen das ganze System
01

Dynamic Type ist die Quelle der Wahrheit.

Jeder Token wird über UIFontMetrics an eine iOS-TextStyle gebunden und skaliert mit den System­einstellungen. Keine harten pt-Werte mehr für UI-Text. Ausnahme nur für display-Numerik (Timer), die ist container-relativ.

02

Zehn Tokens, niemals mehr.

display · title · screenTitle · section · body · bodyEmphasis · bodyItalic · caption · micro · eyebrow. Wenn ein Screen einen elften will, ist der Screen falsch entworfen, nicht das System.

03

Serif spricht, Sans steuert.

Newsreader (300, 400 italic) für alles Erzählende: Titel, Numerik, Akzente. Geist (400, 500) für alles Funktionale: Labels, Werte, CTAs, Captions. Italic ist Farbe, kein Style: nur via <em> oder bodyItalic in var(--hl).

Regel: Die Display-Numerik (Timer-Countdown, Dial-Value) wird nicht als feste pt-Größe gesetzt, sondern als ringDiameter × 0.32 berechnet — mit einem Floor von 56pt und einem Ceiling von 120pt. So passt sie auf jedem Gerät, ohne dass ein Magic-Number-Wert pro Screen-Class nötig wäre.

Verhalten unter Accessibility

xSmall → AX5

Apple definiert 12 Größen: 7 normale (xSmall → xxxLarge) und 5 Accessibility-Größen (AX1 → AX5). Unsere Regeln pro Token:

xSmall
~82 %
Large (Default)
100 %
xxxLarge
123 %
AX1
~135 %
AX3
194 %
AX5
235 %
Alle Tokens skalieren bis AX5. Ausnahme: .display (Timer-Numerik) cappt bei AX1 — sonst sprengt sie den Ring. Stattdessen wird ab AX2 die Numerik unter den Ring verschoben und das Ring-Inneres bleibt leer. Erinnerungs-Sheets, Player-Sheet und Settings-Listen wechseln ab AX3 von Multi-Column-Layouts zu gestapelten Layouts (.dynamicTypeSize(.x) ⇒ VStack).
Bold Text Setting: Wenn der User „Fett gedruckter Text" aktiviert, verschiebt iOS die System-Fonts um einen Weight nach oben. Unsere Custom-Fonts ignorieren das standardmäßig — also müssen wir @Environment(\.legibilityWeight) abfragen und Geist 400 → 500, Newsreader 300 → 400 mappen. Newsreader 400 bleibt 400 (kein 500-Weight verfügbar).

Token-Definition TextStyle.swift 
import SwiftUI

enum TextStyle {
    case display       // container-relativ — siehe .displayFont(in:)
    case title         // .largeTitle  · Newsreader 300
    case screenTitle   // .title1      · Newsreader 300
    case section       // .title3      · Newsreader 300
    case body          // .body        · Geist 400
    case bodyEmphasis  // .body        · Geist 500
    case bodyItalic    // .body        · Newsreader 400 italic
    case caption       // .subheadline · Geist 400
    case micro         // .caption2    · Geist 400
    case eyebrow       // .caption2    · Geist 400 UPPER tracked

    var textStyle: Font.TextStyle {
        switch self {
        case .display, .title:           return .largeTitle
        case .screenTitle:                return .title
        case .section:                    return .title3
        case .body, .bodyEmphasis, .bodyItalic:
                                          return .body
        case .caption:                    return .subheadline
        case .micro, .eyebrow:            return .caption2
        }
    }

    var baseSize: CGFloat {
        switch self {
        case .display:       return 88   // override per Container
        case .title:         return 30
        case .screenTitle:   return 26
        case .section:       return 20
        case .body, .bodyEmphasis, .bodyItalic:
                              return 17
        case .caption:       return 14
        case .micro, .eyebrow: return 11
        }
    }

    var fontName: String {
        switch self {
        case .display, .title, .screenTitle, .section:
                              return "Newsreader-Light"
        case .body, .caption, .micro, .eyebrow:
                              return "Geist-Regular"
        case .bodyEmphasis: return "Geist-Medium"
        case .bodyItalic:   return "Newsreader-Italic"
        }
    }

    var tracking: CGFloat {
        switch self {
        case .title, .screenTitle: return -0.4
        case .eyebrow:             return  2.4   // 0.22em @ 11pt
        default:                   return  0
        }
    }

    var uppercase: Bool { self == .eyebrow }
}
View-Modifier TextStyle+View.swift 
extension View {
    func textStyle(_ style: TextStyle, monospacedDigits: Bool = false) -> some View {
        modifier(TextStyleModifier(style: style, monoDigits: monospacedDigits))
    }
}

struct TextStyleModifier: ViewModifier {
    let style: TextStyle
    let monoDigits: Bool
    @Environment(\.legibilityWeight) private var legibility

    func body(content: Content) -> some View {
        let name = effectiveFontName()
        let font = Font.custom(name, size: style.baseSize, relativeTo: style.textStyle)

        content
            .font(monoDigits ? font.monospacedDigit() : font)
            .tracking(style.tracking)
            .textCase(style.uppercase ? .uppercase : nil)
    }

    private func effectiveFontName() -> String {
        // Bold Text Setting → Weight um einen Schritt rauf
        if legibility == .bold {
            switch style {
            case .body, .caption, .micro, .eyebrow: return "Geist-Medium"
            case .bodyEmphasis:                     return "Geist-SemiBold"
            case .display, .title, .screenTitle, .section:
                                                     return "Newsreader-Regular"
            case .bodyItalic:                       return "Newsreader-Italic"
            }
        }
        return style.fontName
    }
}
Display-Numerik · container-relativ DisplayNumeral.swift 
// Statt fixe 100pt für die Timer-Numerik: ringDiameter × 0.32, gefloort/geceilt.
// Bei AX2+ wird die Numerik unter den Ring gerendert (siehe Timer-Spec).

struct DisplayNumeral: View {
    let text: String
    let ringDiameter: CGFloat
    @ScaledMetric(relativeTo: .largeTitle) private var scale: CGFloat = 1
    @Environment(\.dynamicTypeSize) private var dynType

    var body: some View {
        Text(text)
            .font(.custom("Newsreader-Light",
                          size: cappedSize,
                          relativeTo: .largeTitle))
            .monospacedDigit()
            .kerning(-1.5)
            .lineLimit(1)
            .minimumScaleFactor(0.7)
    }

    private var cappedSize: CGFloat {
        let raw = ringDiameter * 0.32
        let floored = max(raw, 56)
        let ceiled = min(floored, 120)
        // Bei AX1 noch leicht hochskalieren, danach cappen
        return dynType >= .accessibility2 ? ceiled : ceiled * scale
    }
}

Acceptance-Checkliste

Für Claude Code

Wenn alle Punkte abgehakt sind, ist das System fertig migriert. In Reihenfolge — Punkt 1 erst, dann Punkt 2, dann Punkt 3 … kein Parallelarbeiten, sonst bricht das Projekt zwischen den Schritten.

1 · Schriften registriert. Newsreader-Light, -Regular, -LightItalic, -Italic und Geist-Regular, -Medium, -SemiBold liegen im Projekt und sind in Info.plist unter UIAppFonts aufgeführt. Debug-Print zeigt sie im Log.
2 · TextStyle.swift existiert. Enum mit den 10 Cases, fontName-, baseSize-, textStyle-, tracking- und uppercase-Properties wie im Snippet oben.
3 · .textStyle(_)-Modifier ersetzt alle .font(_)-Aufrufe. Globale Suche: kein .font(.system(size:, kein .font(.custom( mehr in View-Code. Einzige Ausnahmen: DisplayNumeral.swift und vielleicht ein Mono-Helper für Code.
4 · Alte Style-Namen entfernt. .timerCountdown, .playerTitle, .bodyPrimary, .bodySecondary, .listSubtitle, .editLabel usw. sind gelöscht — nicht deprecated, gelöscht. Compiler-Fehler weisen automatisch auf alle Restbestände hin. Migration-Map oben ist die Übersetzung.
5 · Display-Numerik ist container-relativ. Timer-Countdown, Timer-Running und Dial-Value rendern über DisplayNumeral(text:, ringDiameter:). Keine hardcoded 100/64/62pt-Werte mehr. Bei AX2+ verschiebt sich die Numerik unter den Ring.
6 · Sekundärfarbe via .foregroundStyle(.secondary), nicht via Token. Wenn ein Text sekundär wirken soll, bleibt der Token gleich — nur die Farbe ändert sich. Niemals .body+kleinere Schrift, um „weniger wichtig" zu sagen.
7 · iPhone-SE-2022-Smoketest. Alle Haupt-Screens (Timer-Idle, Timer-Running, Player, Library, Settings, Dialog-Sheets) auf SE 2022 (375 × 667) durchklicken. Kein Truncate auf primärem Titel, kein Overflow, Tab-Bar nicht von Inhalten verdeckt.
8 · Dynamic-Type-Smoketest auf AX3. Mit Environment(\.dynamicTypeSize, .accessibility3) in der Preview jeden Hauptscreen rendern. List-Rows wechseln zu VStack, Settings-Layout bleibt scrollbar, Timer zeigt Numerik unterhalb des Rings, Bottom-Sheets werden nicht von der Tab-Bar überdeckt.
9 · Bold-Text-Setting honoriert. Mit Environment(\.legibilityWeight, .bold) in Preview: Geist-Text wird Medium, Newsreader-Light wird Regular. Italic bleibt Italic. Keine fetten serifen Titel — Newsreader 500 existiert nicht im Asset-Set.
10 · Typography-Reference-Screen aktualisiert. Der bestehende Debug-Screen („Typography Reference") zeigt jetzt nur noch die 10 Tokens, jeweils mit Name, Sample, Dynamic-Type-Basis und Family/Weight. Slider zum Schalten von Dynamic-Type-Größe (xS, L, AX1, AX3, AX5) eingebaut.
